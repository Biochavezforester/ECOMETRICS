## I Have a Question
If you have questions about how to use the software, please open an Issue on GitHub with the label `question`.

## I Want To Contribute

### Reporting Bugs
Bugs are tracked as GitHub issues. When you are creating a bug report, please include as many details as possible:
* Use a clear and descriptive title.
* Describe the exact steps which reproduce the problem.
* Provide specific examples or datasets (if possible) to demonstrate the issue.

### Suggesting Enhancements
Enhancement suggestions are tracked as GitHub issues. When creating an enhancement suggestion, please provide a clear and detailed explanation of the feature and why it would be useful for the ecology or forestry community.

### Your First Code Contribution
1. Fork the repository.
2. Clone your fork locally.
3. Install dependencies using `pip install -r requirements.txt`.
4. Create a new branch (`git checkout -b feature/my-new-feature`).
5. Make your changes and ensure tests pass by running `pytest`.
6. Commit your changes (`git commit -am 'Add some feature'`).
7. Push to the branch (`git push origin feature/my-new-feature`).
8. Create a new Pull Request.
